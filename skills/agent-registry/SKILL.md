# Agent Registry

## Overview

This skill reads `~/.claude/conductor-registry.yaml` — generated by `/conductor:scan-agents` — to surface available agents with their descriptions and configured models, and to recommend the best match for a given task.

## Usage

1. Check whether `~/.claude/conductor-registry.yaml` exists.
2. If it exists, read the file and present the agent table: `name | description | model` (model column only when present in frontmatter).
3. Match the task description against agent descriptions and recommend the top 1–3 candidates with a brief rationale.

## Registry Format

```yaml
generated: YYYY-MM-DD
sources:
  - path: ~/.claude/agents/
    count: N
  - path: .claude/agents/
    count: N
agents:
  - name: agent-name
    description: "Role description extracted from frontmatter"
    model: claude-haiku-4-5  # optional — only present if declared in frontmatter
```

## Recommendation Logic

When recommending an agent:
- Match task domain keywords against the `description` field.
- Prefer agents with an explicit `model` declaration if the task is cost-sensitive (lighter models for narrow tasks).
- If multiple agents match, list them ranked by specificity of their description.
- If no agent clearly matches, say so and suggest running `/conductor:dispatch <task>` to generate a generic structured prompt instead.

## If Registry Is Absent

When `~/.claude/conductor-registry.yaml` does not exist:
- Inform the user that the registry has not been built yet.
- Suggest running `/conductor:scan-agents` to generate it.
- Do not attempt to guess available agents from memory.

## Keeping the Registry Fresh

The registry is a static snapshot. Re-run `/conductor:scan-agents` after:
- Installing new Claude Code agents (user-level or project-level)
- Modifying agent frontmatter (description or model)
- Adding a new project with its own `.claude/agents/` directory
